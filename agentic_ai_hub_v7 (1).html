Top Cost Reduction Strategies
Model routing by complexity
Up to 10x savings
Route simple tasks to cheap fast models. Reserve expensive models for genuine complex reasoning. A routing call costs fractions of a cent and can cut 60–80% of total LLM spend.
1
Classify request complexity at intake using a cheap model
2
Route: simple → cheapest model tier, complex → mid tier, critical → premium tier
3
Track per-route quality scores; tune thresholds with eval data
Prompt caching
Up to 90% input savings
Static content (system prompt, tool schemas, reference docs) can be cached. Cached input costs ~90% less than regular input. Critical for agents with long system prompts.
1
Identify static content: system prompt, tools, reference docs
2
Add cache_control: ephemeral to those blocks in API calls
3
Monitor cache_read_input_tokens in API response to verify savings
Batch API for async work
50% cost reduction
Any work that doesn't need real-time response (evals, reports, bulk analysis) can use the Batch API. Most providers offer ~50% discount for async processing.
1
Identify tasks tolerant of up to 24-hour turnaround
2
Submit as JSONL via Batch API, poll for completion
3
Retrieve, validate quality, process downstream
Embedding cost control
90%+ ingestion savings
Embedding costs dominate ingestion pipelines. Use smaller models for bulk ingestion, larger for quality-critical queries. Cache query embeddings for repeated queries.
1
Use a small/cheap model for document ingestion (bulk)
2
Use a higher-quality model for query embeddings at inference
3
Cache query embeddings in Redis with short TTL for popular queries
RAG context compression
40–60% input savings
Retrieved chunks are often the biggest token cost per request. Compress before inserting: extract only the relevant sentences, not the full chunk.
1
After retrieval, run contextual compression: extract relevant sentences only
2
Set a result_budget token limit per chunk in your retrieval system
3
Measure compression ratio vs quality impact on your eval set
Tool result compression
Reduce loop cost 50%+
Tool results grow the context window every iteration — this compounds across all loop calls. Summarize tool outputs before reinserting them into the context.
1
After each tool call, summarize result with a cheap model or regex
2
Insert the summary (not raw output) back into the agent context
3
Track context growth per iteration — alert if >500 tokens/step

Pre-Production Agent Checklist
0 / 12
Context management: Rolling summarization implemented when context exceeds 80% of limit
Iteration budget: Hard max on LLM calls per invocation is set and enforced
Tool error handling: Every tool call has error handling and retry with exponential backoff
Model version pinned: Using exact model string, not a floating alias like "latest"
Golden eval set: At least 50 representative test cases with verified expected outputs
Output validation: Schema validation on all structured outputs with auto-retry on failure
Safety evals: Adversarial test suite passing before every deploy
Observability: Traces, spans, and tool call logs going to an observability platform
Cost limits: Per-request token budget enforced; cost anomaly alerts configured
Rollback plan: Previous model version retained; rollback procedure documented and tested
Secrets management: API keys in secrets manager, not hardcoded or in plaintext env vars
PII handling: Confirmed no PII in logs, traces, or episodic memory store
RAG System Checklist
0 / 7
Chunking validated: Chunk size and overlap tested on representative documents for your domain
No-context baseline: Retrieval value delta measured (RAG score minus parametric-only score)
Faithfulness monitoring: LLM-judge faithfulness scoring on sampled production traffic
Index freshness policy: TTL and refresh schedule defined and automated
Embedding model locked: Model version tracked per record; re-embedding plan exists for upgrades
Hybrid search: Both dense vector and sparse BM25 retrieval enabled and fused
Query latency SLA: Retrieval p95 latency measured and within acceptable threshold
Multi-Agent System Checklist
0 / 6
Context isolation: Sub-agents receive task brief only, not the orchestrator's full history
Agent identity: Each agent has a defined role, tool scope, and permission level
Shared state store: Inter-agent state uses external store (Redis/DB), not shared context windows
Prompt injection defense: Tool outputs sanitized before reinsertion into any agent context
Escalation path: Defined conditions that route to human review before irreversible actions
System-wide iteration cap: Total LLM calls across all agents in one invocation is bounded
